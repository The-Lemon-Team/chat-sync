## Session: Sync + WebSocket (2026-06-11)

### SyncService
- Incremental sync via `minId`, full history via `offsetId` pagination
- Media download → `MEDIA_STORAGE_PATH/{chatForkId}/{messageId}/`
- Respects `OFFLINE_ONLY` — skip Telegram fetch
- `@Cron` every minute for `SCHEDULED` forks

### ChatGateway
- Socket.IO namespace `/chat`
- Events: `join`, `leave`, `message:new`, `message:updated`, `sync:progress`
- Rooms: `fork:{chatForkId}`

### ChatFork API
```
GET/POST /chat-forks
GET /chat-forks/:id/messages
POST /chat-forks/:id/sync
```

## Session: Frontend (2026-06-12)

### Frontend stack
```

├── src/stores/     account, chat, safety

├── src/db/         dexie.ts
├── src/components/ SafeModeBanner, ForkSidebar, AddChatModal, ChatPanel

```

### Safe Mode design
- `OutboundGuard.evaluate(action, chatFork, options)`
- Actions: `send_telegram`, `sync_pull`, `sync_push`
- Default: messages saved as `OFFLINE_ONLY`, never sent to TG
- User must unlock LIVE explicitly

### Manual chat selection
- No auto-import on login
- `GET /telegram/accounts/:id/dialogs` lists available
- User picks one → `POST /chat-forks` with default `MANUAL`

### API tests
```

✓ GET /chat-forks
```
