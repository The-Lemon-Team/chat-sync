# Memory — Chat-Sync

Папка для **логов работы**, **сводки проекта** и **Kanban-доски** (Obsidian).

## Структура

| Файл | Назначение | Когда обновлять |
|---|---|---|
| [`LOG.md`](LOG.md) | **Актуальный лог** — последние сессии (≤ ~80 строк) | После каждой сессии разработки |
| [`SUMMARY.md`](SUMMARY.md) | **Живая сводка** — что построено, как запустить, что дальше | При завершении milestone |
| [`kanban.md`](kanban.md) | **Доска задач** (Obsidian Kanban plugin) | При смене статуса задачи |
| [`history/`](history/) | **Архив** — старые записи из LOG.md | Когда LOG.md переполнен |

## Workflow (лучшие практики)

```
1. Работа → дописать запись в LOG.md (сверху, newest-first)
2. Milestone закрыт → обновить SUMMARY.md + kanban.md + AGENTS.md
3. LOG.md > ~80 строк → перенести старые блоки в history/YYYY-MM/YYYY-MM-DD-topic.md
4. В LOG.md оставить ссылку: «Ранее: history/…»
```

## Формат записи в LOG.md

```markdown
## YYYY-MM-DD — Краткий заголовок сессии

**Цель:** …
**Сделано:**
- …
**Коммиты:** `hash` — message
**Следующий шаг:** …
```

## Obsidian

1. Открыть vault = корень репозитория (или symlink на `memory/`)
2. Установить community plugin **Kanban**
3. Открыть `memory/kanban.md`

## Связь с AGENTS.md

- `AGENTS.md` — инструкции для AI-агентов (роли, команды, чеклист)
- `memory/` — человекочитаемая история и планирование
- При расхождении **kanban.md** и **AGENTS.md** — синхронизировать оба
